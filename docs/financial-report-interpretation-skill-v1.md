# 财务报告解读 Skill V1

状态：已作为 repo instruction 实现；runtime Skill 集成待完成。

这份 instruction 定义 Financial Results Report 的中文叙事层。中文报告是面向人的主版本；英文只保留在必要的文件类型、KPI 缩写、JSON 字段和公式 id 中。

## 输入

中文易读版按三层输入生成。第一层输入是必需的；第二层和第三层如果尚未生成，就明确显示“未接入”，不能用模型记忆补齐。

### 第一层输入：`financial_report_pack.json`

这个 pack 由确定性 Python 流水线生成，包含：

- 公司信息和来源清单。
- 年报 / 10-K / 20-F 基准。
- 年度与季度事实。
- 财务指标和估值指标。
- 诊断发现。
- 财务异常解释 notes。
- 重大事项扫描。
- 验证与交叉检查结果。
- 缺失事实和人工复核标记。

### 第二层输入：`official_report_evidence_pack.json`

这个 pack 由 Official Report Evidence Agent 生成，主要包含：

- 第一层问题的官方文件回答。
- 官方 filing / report 中的重要叙事、战略变化、治理事项和公司特有 KPI。
- 官方文件事实、管理层解释、我们的证据裁判和仍未知事项。
- 第二层无法回答、需要转交第三层或更外部研究的问题。

### 第三层输入：`management_communication_pack.json`

这个 pack 由 Management Communication Agent 生成，主要包含：

- earnings call transcript、Q&A、shareholder letter、investor deck、investor day、management interview、official newsroom / product announcement 等管理层沟通材料。
- 管理层对前两层问题的解释。
- 回答质量：是否有数字、是否具体、是否只是方向性表述、是否回避、是否与前两层冲突。
- 新管理层叙事、战略承诺、Q&A 压力主题和后续 watchlist。

## 硬规则

- 第一层、第二层、第三层都采用 question-led workflow，同时保留有限的 discovery-led insight。
- 不重新计算数字。
- 不改公式。
- 不添加 pack 里没有的事实。
- 没有 `management_communication_pack.json` 时，不在本报告中使用业绩电话会文字稿。
- 不把第三方数据作为财务事实来源。
- 不把每一份 6-K / 8-K / proxy / prospectus 都写成独立章节。
- 只有当 `material_event_scan.events` 标记为重大时，才把非核心文件放进正文。
- 异常指标需要解释时，使用 `financial_investigation_notes`。
- 必须清楚区分：官方文件事实、管理层解释、基于数据的推断、仍未知。
- 近似公式必须标注为近似值。
- 表格必须有明确比较优势才使用：例如跨年度、跨季度、跨多个指标并列比较。若只是 2-4 个数字、一个结构拆分或一条现金流桥，优先用短段落或简短 bullet，避免为了“看起来结构化”而让报告变凌乱。

## 写作顺序

1. 先从年报 / 10-K / 20-F 建立主线。
2. 再用最新季度或中期数据验证、削弱或改变这条主线。
3. 第一层先回答固定财务问题：
   - 公司靠什么赚钱。
   - 增长来自哪里。
   - 利润能不能变成现金。
   - 主要风险是什么。
   - 哪些数字或会计口径让人不舒服。
4. 第一层再做数据洞察与异常发现：只从硬数字里发现反常组合，例如收入增长但经营利润下降、增量经营利润率为负、CFO 受营运资本顺风帮助、现金很厚但受限现金比例高、收入组件发生结构变化。这里不能解释动机，只能把问题交给第二层或第三层验证。
5. 第二层回答官方文件能解释什么，并登记官方文件里的重要叙事、战略变化、治理和风险信号。
6. 第三层回答管理层沟通能解释什么，并判断回答质量、Q&A 压力和管理层叙事变化。
7. 只有扫描器发现重大事项时，才加入重大事项。
8. 对异常指标，按这个顺序写：
   - 问题是什么：用一句话概括第一层留下的异常和关键数字。
   - 官方文件能说明什么：只写官方文件实际解释了什么、定义了什么、披露了什么。
   - 仍未解决 / 风险：统一放入官方解释覆盖不了的部分、证据缺口、风险边界和我们的证据裁判。
9. 把财务质量判断和估值判断分开。
10. 最后列出未解决事实、第三层待追问事项和人工复核点。

## 语气

用简洁中文写给价值投资者。报告要帮助读者决定下一步应该回到哪一份原始文件继续读。

不要写得比证据更确定。管理层解释不是事实；模型推断也不是事实。

## 必要标签

适用时使用这些中文标签：

- 所有者收益近似值：没有独立估算维护性资本开支，只能作为近似线索。
- 无杠杆 ROIC 近似值：投入资本使用融资侧近似口径。
- 增量 ROIC 近似值：当投入资本很小、为负或现金很重时，噪声较大。
- 自由现金流定义：V1 使用官方事实中的经营现金流减资本开支。
- 重大事项扫描：不是对所有非核心 filings 的完整摘要。

## 输出章节

推荐章节：

1. 一页结论与动作卡片。
2. 第一层：核心指标与红旗诊断（财务证据层）。
   - 核心判断。
   - 主要问题。
   - 主要财务指标。
   - 数据洞察与异常发现。
3. 第二层：官方文件解释与叙事发现（官方报告层）。
   - 第一层问题的官方文件回答。
   - 官方文件中的重要叙事与战略变化。
     - 第二层按文件类型覆盖 official reports / filings：20-F、10-K、formal 10-Q、material 6-K / 8-K、prospectus、proxy / AGM materials、notes、MD&A / OFR、audit / ICFR / CAM、debt / tax / VIE / accounting policy disclosures。
     - Earnings release 是例外：第一层可以为计算季度数字抽取结构化财务数据；第二层读取 release 的管理层叙事、CEO / CFO quote、non-GAAP 解释和季度口径；第三层不读取 earnings release。
     - Shareholder letter、investor presentation / earnings deck、investor day deck、official newsroom / product announcement 归第三层。
     - 正文不要逐条展示“官方线索”；证据追踪放在 evidence report / JSON。
     - 每条叙事优先写成：文件内容、投资读法、仍需验证。
     - 投资读法要比普通摘要更强：说明这条叙事如何改变商业模式、利润率、现金、资产负债表或风险判断。
4. 第三层：管理层沟通与叙事裁判（管理层沟通层）。
   - 前两层留下的问题：管理层怎么解释。
   - 管理层新叙事与战略变化。
   - Q&A 压力与回答质量。
   - 若第三层 pack 未接入，必须明确写“未运行第三层”，只列后续应追问的问题。
5. 后续跟踪清单。
6. 附录：来源、口径与复核点。

## 失败模式

如果 pack 缺少年度事实或年报基准，就明确说明报告不完整，并列出缺失字段。不要用模型记忆或网络数据补空白。
